Future-Self Career Chatbot — Evaluation Pipeline

BSc Thesis — Offline demo report. Generated automatically by python -m eval_pipeline.run_demo.

1. Method Summary

Students completed a ~30-minute future-self career chatbot study: pre-survey → career selection → role-play conversation with an LLM acting as their future self (phaseC) → post-survey.

This pipeline answers: Can an LLM persona/digital-twin, built from a participant's questionnaire, reproduce that participant's post-chat self-report ratings — and how do design choices (persona depth, prompt structure) affect agreement with human ratings?

Design: For each participant, we build a persona prompt at a given depth (D0–D2), feed it the participant's actual phaseC transcript, and ask it to fill in the post-survey battery as that participant. Each Likert item is elicited via SSR (Semantic Similarity Rating): the persona generates a free-text response; that response is embedded; cosine similarity to one anchor statement per scale point is computed; softmax → expected value (continuous rating) and mode (discrete). Agreement between predicted and actual post ratings is then quantified.

We ablate across:

  • Persona depth: D0 (demographics) → D1 (+psychometrics+career) → D2 (+own words)
  • Prompt structure: structured / narrative / interview framing
  • D3 leakage probe: includes pre-outcome scores — shown as an inflation check only
Each (participant × config) is rated k=5 times; inter-run SD captures persona stability.

Embedder note: The offline demo uses a deterministic char n-gram hashing vectorizer (no network, no model download). Swap in SentenceTransformerEmbedder("all-MiniLM-L6-v2") from eval_pipeline/embedder.py for real use.

2. Outcome Instruments

  • Continuity (FSCS, 3 items): "How similar/connected/care..." — 1=Not at all, 7=Completely. Mean of 3 items.
  • Vividness (4 items): viv_clear/tangible/detail/felt — 1=Strongly disagree, 7=Strongly agree. Mean of 4.
  • Closeness (IOS, 1 item): ios_post — 1=Completely separate, 7=Completely overlapping.
  • Manipulation Checks (3 items): mc_style/scene/understand — transcript-observable. Mean of 3.
  • Persuasiveness: NOT collected by the app. Slot reserved; labeled synthetic-only.

D3 is reported here as a methodological check only and is excluded from all substantive conclusions.

5. Metric Definitions

MAE: Mean Absolute Error between predicted and actual continuous ratings.
Spearman ρ: Rank correlation between predicted and actual ratings (level-agreement).
QWK: Quadratic-weighted Cohen's κ — penalises large disagreements more than small ones.
ICC(2,1): Intraclass Correlation Coefficient, two-way random, absolute agreement, single measures.
Adjacent Accuracy: Proportion of predictions within ±1 of the actual rounded rating.
Bland-Altman: Systematic bias (mean difference) and 95% limits of agreement (±1.96 SD of differences).
Δ-correlation: Spearman between (predicted_post − actual_pre) and (actual_post − actual_pre). Measures whether the model captures change from baseline. Expected to be lower than level-correlation.
Inter-run SD: Mean per-participant SD across k=5 repeated runs at the same config (persona stability).
FSCS test-retest context: Literature ceiling ≈ .66 (Ersner-Hershfield, 2009). No test-retest data available in this study → normalized accuracy not computed.

6. Limitations

  • Synthetic data only: All results shown here use synthetic sessions with planted latent truth. Real session exports have not been collected yet.
  • No test-retest data: Park (2019) normalized accuracy cannot be computed; FSCS test-retest r≈.66 cited as context only.
  • Single judge model: Only one LLM (FakeLLM here; claude-sonnet-4-6 in production) acts as judge. No inter-model agreement computed.
  • SSR stub embedder: The offline demo uses a char n-gram hashing vectorizer as a stand-in. Real agreement requires sentence-transformers or equivalent.
  • Persuasiveness not collected: The app does not export a persuasiveness outcome. The slot is reserved but empty.
  • Small N: Default N=24 synthetic participants. Results are descriptive only; no inferential testing.
  • Model/temperature fixed: Model is constrained to claude-sonnet-4-6 (matches the app). Temperature and seed are varied instead of model; this study does not implement cross-model comparison.
